# PR Body Sync Gate

## 1. 목적

구현 후 팀 이해·오해 점검 결과가 PR 댓글에만 남고 PR 본문의 전용 영역은 비어 있는 상태를 방지한다.

이 문서는 `.codex/skills/coffee-order-issue-loop/SKILL.md`의 Human·팀원 답변 확인 단계를 보완하는 필수 규칙이다.

## 2. 권위 있는 기록 위치

- 구현 후 팀 이해·오해 점검의 권위 있는 최종 기록은 PR 본문이다.
- PR 댓글은 검토 과정, 긴 코드 근거, 변경 이력, 추가 설명을 남기는 보조 기록이다.
- 댓글 작성만으로 PR 본문의 빈 항목을 대체해서는 안 된다.
- 댓글과 PR 본문이 충돌하면 PR 본문을 최신 상태로 동기화한 뒤 다시 확인한다.

## 3. Human 답변 보존

- Human 작성자와 리뷰어가 작성한 답변은 PR 본문의 기존 답변 칸에 원문 그대로 보존한다.
- Codex와 ChatGPT는 Human 답변을 수정, 요약, 재작성하거나 대신 작성하지 않는다.
- 표현이 미숙하거나 일부 내용이 부족해도 Human 답변 원문을 AI 문장으로 교체하지 않는다.
- 보완이 필요하면 Human 답변 아래의 `잘 모르겠는 부분` 또는 별도의 AI 대조 표를 사용한다.

## 4. AI 코드 대조 결과 기록

작성자와 필요한 리뷰어가 답변한 뒤 Codex 또는 ChatGPT는 최신 Head의 실제 코드와 테스트를 다시 읽고 다음을 PR 본문에 기록한다.

- 코드와 일치하는 부분
- 보완 설명이 필요한 부분
- 확인된 미검증·잔여 위험
- 중대한 오해 후보

기록 위치는 PR 본문의 `### AI 코드 대조와 보완 설명` 표다.

- LOW는 Q1~Q3을 모두 기록한다.
- MEDIUM은 Q1~Q5를 모두 기록한다.
- HIGH는 실제 생성된 Q1~Qn과 필요한 리뷰어 행을 모두 기록한다.
- 각 행에는 `코드와 일치하는 부분`, `보완 설명`, `중대한 오해 여부`를 빠짐없이 기록한다.
- 대조 결과를 댓글에 먼저 남겼다면 동일한 최종 결과를 PR 본문 표에 반영한다.
- 댓글 링크만 남기고 PR 본문 표를 비워 두면 완료가 아니다.

## 5. 상태 처리

이해·오해 점검에는 다음 상태만 사용한다.

- `확인 완료`
- `보완 설명 필요`
- `보완 설명 완료`
- `차단 필요`

`PASS | HOLD`는 Contract Traceability의 코드·계약 검증에만 사용하고 Human 이해 점검에는 사용하지 않는다.

Codex와 ChatGPT는 보완 설명과 중대한 오해 후보를 기록할 수 있다. 중대한 오해 여부의 최종 판단은 Human 리뷰어가 한다.

## 6. Human 최종 판단 보호

PR 본문의 `### Human 최종 판단` 영역은 Human 전용이다.

Codex와 ChatGPT는 다음 항목을 대신 수행하지 않는다.

- Human 전용 체크박스 선행 체크
- 최종 상태 확정
- 보완 설명 확인 기록을 Human 대신 작성
- 중대한 오해 없음 최종 선언
- 최종 승인 또는 Merge 결정

AI가 수행할 수 있는 작업은 실제 코드 대조 결과와 보완 설명을 PR 본문의 전용 표에 기록하는 것까지다.

## 7. 본문 갱신 실패

PR 본문 수정 권한이나 도구 문제로 본문을 갱신할 수 없으면 댓글만 작성하고 완료로 보고하지 않는다.

다음 상태로 중단한다.

```text
BLOCKED: PR BODY SYNC REQUIRED
```

보고 내용:

- 비어 있거나 갱신하지 못한 PR 본문 영역
- 댓글에 임시 기록한 내용과 링크
- 본문 갱신이 실패한 이유
- Human 또는 권한 있는 도구가 수행해야 할 작업

## 8. 본문 갱신 후 재확인

PR 본문을 갱신한 뒤 반드시 실제 PR 본문을 다시 읽고 다음을 확인한다.

- Human 답변 원문이 보존됐는가
- 위험도에 필요한 Q1~Qn 행이 모두 존재하는가
- `AI 코드 대조와 보완 설명` 표가 비어 있지 않은가
- 문항별 보완 설명과 중대한 오해 여부가 기록됐는가
- 비담당 리뷰어 요약과 AI 대조 결과가 필요한 위험도 기준을 충족하는가
- 댓글과 PR 본문의 상태가 충돌하지 않는가
- Human 전용 체크박스를 AI가 선행 체크하지 않았는가
- 최신 Head Commit과 CI 정보가 일치하는가

하나라도 충족하지 못하면 이해·오해 점검 완료로 보고하지 않는다.

## 9. 완료 판정

다음 조건을 모두 만족해야 이해·오해 점검의 AI 기록 단계가 완료다.

- PR 본문에 Human 답변 원문 보존
- PR 본문의 AI 코드 대조 표에 필요한 모든 문항 작성
- 문항별 보완 설명과 중대한 오해 후보 기록
- 필요한 비담당 리뷰어 이해 요약 기록
- 댓글과 본문 상태 동기화
- Human 최종 판단 영역 미선행 처리
- 최신 Head와 CI 정보 확인
- 갱신된 PR 본문 재조회 완료

댓글에만 결과가 있고 PR 본문의 전용 영역이 비어 있으면 완료가 아니다.
