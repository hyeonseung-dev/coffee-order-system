# Harness Logs

## 1. 목적

`logs` 디렉터리는 테스트, 검증, 의미 있는 트러블슈팅과 Human 판단의 근거를 보존한다.

로그는 하네스 형식을 맞추기 위한 문서가 아니라 다음 질문에 답하기 위한 증거다.

- 무엇을 테스트했는가
- 테스트가 무엇을 보장하는가
- 무엇을 검증하지 못했는가
- 어떤 문제가 발생했는가
- 원인을 어떻게 좁혔는가
- 어떤 해결을 선택했고 왜 선택했는가
- 같은 문제의 재발을 어떻게 막았는가

실행하지 않은 명령을 실행한 것으로 기록하지 않고, 실패를 성공으로 바꾸지 않는다.

## 2. 기본 기록 위치

일반적인 구현은 Issue와 PR 본문을 기본 기록 위치로 사용한다.

별도 로그가 필요한 경우:

```text
logs/issues/issue-{번호}/attempt-log.md
```

검증 결과를 누적할 필요가 있을 때:

```text
logs/verification-log.md
```

모든 작은 Issue에 별도 Attempt Log 생성을 강제하지 않는다.

## 3. 별도 로그를 남길 가치가 있는 경우

- 설계 가정과 실제 동작이 달랐음
- 트랜잭션·롤백·데이터 정합성 문제
- 동시성 경쟁 조건과 중복 처리 문제
- 성능 목표 미달
- 캐시·메시징·DB 장애 또는 환경 차이
- 로컬과 CI 결과 차이
- 여러 해결 방법을 비교함
- 재발 방지 테스트를 추가함
- Issue 또는 ADR을 변경함
- Human이 중요한 선택을 수행함

단순 오타, import 누락, 명확한 Mock 설정 수정은 별도 로그를 강제하지 않는다.

## 4. 기록 시점

### 문제 발생 시

- 증상과 재현 조건
- 실패 명령과 핵심 로그
- 확인된 사실
- 현재 가설

### 해결 시

- 실제 원인
- 시도한 방법과 결과
- 최종 해결
- 실패한 접근
- 재발 방지

### PR 검토 시

- 임시 우회인지 근본 해결인지
- 테스트로 재현과 재발 방지가 가능한지
- 운영 환경에서도 유효한지
- Issue·ADR·문서 갱신이 필요한지

## 5. 테스트와 검증 기록

테스트 개수보다 보장 내용을 기록한다.

```markdown
## 테스트가 보장하는 것

-

## 테스트가 보장하지 않는 것

-

## 실행한 검증

- 명령:
- 결과:
- 직접 검증:
- DB·로그 확인:

## 미검증 항목

-
```

실제 다중 서버, 네트워크 장애, 운영 트래픽처럼 검증하지 못한 범위를 성공한 것처럼 표현하지 않는다.

## 6. Troubleshooting Gate 기록

중요한 문제로 작업이 중단되면 다음 형식을 사용한다.

```markdown
## Troubleshooting Gate

### 발생한 문제

### 재현 조건

### 확인된 사실

### 추정 원인

### 시도한 방법과 결과

### 영향 범위
- 기능:
- 데이터:
- 트랜잭션:
- 테스트:
- 운영:

### 가능한 선택지

### 권장 방향

### 필요한 Human 결정
```

확인된 사실과 추정 원인을 구분한다.

## 7. 최소 운영형 차단 기준

형식적 기록 누락만으로 Push 또는 PR을 차단하지 않는다.

경고 대상:

- 하네스 전용 환경변수 누락
- Attempt 번호와 상태 문자열 누락
- 별도 로그 파일 누락
- PR 선택 항목 누락
- GitHub 상태 보고 실패

차단 대상:

- 필수 하네스 파일 누락
- Bash 문법 오류
- Gradle 테스트 실패
- Gradle 빌드 실패
- 보호 브랜치 직접 Push
- 테스트 삭제·비활성화·약화
- 요구사항·정합성·보안·운영 위험 은폐
- Human Gate 없이 중요한 설계 변경 진행

CI 성공 여부는 PR 본문의 PASS 선언이 아니라 실제 테스트와 빌드 결과로 판단한다.

## 8. Human 기록

Human은 다음과 같은 중요한 판단만 남긴다.

- READY 승인
- ADR 승인 또는 반려
- Troubleshooting Gate 선택
- 보류 또는 범위 변경
- 최종 Merge 여부

ChatGPT나 Codex의 PASS 판단은 Human Merge를 대신하지 않는다.

## 9. Merge 원칙

- ChatGPT Merge 금지
- Codex Merge 금지
- 자동 Merge 금지
- Human만 Merge 가능
