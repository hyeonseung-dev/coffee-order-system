# Human 착수 검토 질문형 양식

## 1. 목적

Human 착수 검토는 담당자가 AI Draft를 그대로 승인하지 않고, 구현 전에 다음 다섯 가지를 확인하는 단계다.

1. 왜 이 작업을 하는가
2. 핵심 흐름과 규칙은 무엇인가
3. 가장 중요한 실패 위험은 무엇인가
4. 무엇을 검증해야 성공이라고 판단할 수 있는가
5. 어디까지 구현하고 무엇을 제외할 것인가

빈 양식에서 처음부터 자유 서술하게 하면 무엇을 검토해야 할지 모호해질 수 있다. 반대로 질문을 지나치게 세분화하면 착수 검토가 사전 시험처럼 변하고 구현·실험보다 문서 작성 비용이 커진다.

따라서 AI가 Issue에 맞는 질문을 먼저 제시하되, **같은 내용을 잘게 쪼개지 않고 원칙적으로 4~6개 질문으로 제한한다.**

---

## 2. 최종 흐름

```text
AI 상세 Draft 작성
→ AI가 핵심 질문 4~6개 작성
→ Human이 자신의 말로 답변
→ ChatGPT가 코드·문서·설계와 대조해 재검증
→ Human·AI 최종 합의
→ Human READY 승인
→ Codex 구현·검증·Draft PR
```

질문에 답했다는 사실만으로 `READY`가 되는 것은 아니다.

```text
Human 질문 답변
+ AI 재검증
+ 최종 합의
+ 완료 조건·검증 기준 확정
+ Human 구현 승인
= READY
```

---

## 3. 역할 구분

### AI Draft 작성자

- 해당 Issue의 실제 목적과 설계 초안을 먼저 제시한다.
- 목적·핵심 흐름·실패 위험·검증 기준·범위를 질문으로 변환한다.
- 일반적인 질문을 기계적으로 복사하지 않는다.
- 세부 명령어를 미리 외우게 하는 질문을 만들지 않는다.
- Human 답변을 대신 작성하지 않는다.

### Human 담당자

- AI Draft와 관련 코드·문서를 읽고 자신의 말로 답한다.
- 확실하지 않은 내용은 `모르겠다`, `검토 필요`, `추가 설명 필요`라고 그대로 적는다.
- AI 제안에 동의하지 않으면 대안과 이유를 적는다.
- 구현 후에는 실제 Diff와 실행 결과를 다시 확인한다.

### ChatGPT 재검증

- Human 답변을 프로젝트 문서, 현재 코드, 선행 Issue, 테스트 기준과 비교한다.
- 잘 이해한 부분과 수정이 필요한 부분을 구분한다.
- 누락된 정합성·장애·보안·성능 위험을 제시한다.
- Human 최초 답변을 덮어쓰거나 대신 작성하지 않는다.

### Codex

- Human 답변, AI 재검증, 최종 합의가 완료된 `READY` Issue만 구현한다.
- 비어 있는 Human 답변을 채우지 않는다.
- 구현 후에는 실제 Diff 기반 이해도 질문만 생성한다.

---

## 4. 질문 수 기준

### 기본 기준

- LOW: 3~4개
- MEDIUM: 4~5개
- HIGH: 5~6개

HIGH라고 해서 질문을 10개 이상 작성하지 않는다. 질문 수보다 다음 내용이 모두 포함됐는지가 중요하다.

- 작업 목적
- 핵심 흐름과 규칙
- 가장 중요한 실패 위험
- 테스트와 직접 검증의 종류
- 대안과 포함·제외 범위
- 미이해·추가 검토 사항

독립적인 위험이 추가로 존재해 7번째 질문이 꼭 필요하다면 그 이유를 남긴다. **7개를 초과하는 착수 질문은 사용하지 않는다.** 세부 명령어, 로그 필드, 장애 재현 절차는 AI Draft의 검증 계획과 구현·PR 단계에 기록한다.

## 6. 좋은 질문과 피해야 할 질문

### 좋은 질문

```text
쓰기와 정합성이 중요한 조회는 왜 Primary에 남기고, 어떤 조회만 Replica로 보낼 수 있나요?
```

- 실제 설계 기준과 위험을 한 번에 확인한다.

```text
복제와 실제 라우팅이 동작했음을 어떤 종류의 DB·로그 증거로 확인해야 하나요?
```

- 검증 방향을 묻되 구체 명령어 암기를 요구하지 않는다.

### 피해야 할 질문

```text
Primary의 server_id를 확인하는 정확한 명령어를 작성하세요.
```

- 구현 전 착수 검토에서 명령어 암기를 요구한다.
- 이런 내용은 검증 계획과 구현 결과에 기록한다.

```text
메뉴 조회 연결 확인, 인기 메뉴 연결 확인, 포인트 충전 연결 확인, 주문 연결 확인 방법을 각각 작성하세요.
```

- 하나의 라우팅 검증 주제를 여러 문항으로 불필요하게 분리한다.

```text
이 Issue를 이해한 대로 모두 작성하세요.
```

- 범위가 너무 넓어 무엇을 검토해야 하는지 알기 어렵다.

## 7. 완료 기준

Human 착수 검토는 다음 조건을 만족해야 완료된 것으로 본다.

- AI Draft에 Issue별 핵심 질문 4~6개가 작성돼 있다.
- Human이 각 질문에 자신의 언어로 답했다.
- 모르는 부분과 추가 검토가 필요한 부분을 숨기지 않았다.
- 목적, 흐름, 핵심 위험, 검증 방향, 포함·제외 범위를 확인했다.
- ChatGPT 재검증과 Human 최종 결정이 기록됐다.
- 위험도, 완료 조건, 수정 범위, 필요한 ADR과 구현 승인이 확정됐다.
