# Course Plan

강의는 과제 제출 가능성을 해치지 않는 범위에서 진행한다.

## 우선순위

1. 개인과제 v0~v2 제출 가능 상태
2. README/docs 문서화

4. 필요한 부분만 실습
5. 최종 프로젝트 대비 확장 학습

## 현재 개인과제 적용 판단

이번 개인과제에서 Kafka는 우선 적용하지 않는다.

이유:

- v0~v2 필수 범위가 먼저다.
- 주문 이벤트 후속 처리는 먼저 Spring Event로 분리한다.
- RabbitMQ는 v3 도전 범위다.
- Kafka는 현재 개인과제보다 최종 프로젝트 대비 학습 가치가 크다.

## 수강 방식

### v0 진행 중

- 강의는 이해 위주
- 실습은 최소화
- 과제 API 구현 우선

### v1 진행 중

- 트랜잭션, 동시성, 에러 처리와 연결되는 부분만 정리
- 실습보다 테스트 작성 우선

### v2 진행 중

- 인기 랭킹, 캐싱, 부하 테스트 강의 우선
- K6 또는 간단한 성능 측정 실습 가능

### v3 진행 중

- RabbitMQ, Kafka, 장애 대응 개념을 도전 과제로 정리
- 구현은 시간이 남을 때만 진행
