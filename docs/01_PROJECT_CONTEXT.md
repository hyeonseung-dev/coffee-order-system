# Project Context

## 프로젝트명

다중 서버 환경을 고려한 포인트 기반 커피 주문 시스템

## 프로젝트 목적

이 프로젝트는 단순 CRUD 구현이 아니라, Java/Spring 백엔드 신입 포트폴리오로 설명 가능한 실무형 개인과제 프로젝트를 목표로 한다.

핵심은 다음과 같다.

- 데이터 정합성
- 트랜잭션
- 동시성 제어
- 인덱싱
- Redis 캐싱
- 테스트
- 문서화
- AI-assisted 개발 프로세스 기록

## 개발 단계

### v0. 필수 API 완성

- 메뉴 조회
- 주문 생성
- 포인트 충전
- 포인트 사용
- 주문 내역 조회

### v1. 트랜잭션 / 동시성 검증

- 포인트 차감 정합성
- 잔액 부족 예외
- 동시 주문 요청
- 비관적 락 또는 낙관적 락 검토
- 테스트로 정합성 검증

### v2. 성능 개선

- 인기 메뉴 조회
- DB 인덱싱
- Redis 캐싱
- 캐싱 전후 성능 비교
- 성능 개선 결과 문서화

### v3. 선택 도전

- MySQL Replication
- Redis Master-Replica
- RabbitMQ 이벤트 분리

단, v3는 선택 도전이며 v0~v2를 우선한다.

## 우선순위

1. 기능 완성
2. 테스트
3. 트랜잭션/정합성
4. 인덱싱/캐싱
5. 성능 비교 문서화
6. 고가용성/메시지 큐 도전

## 하지 않을 것

- MSA
- Kafka 우선 도입
- 과도한 CI/CD
- 과도한 모니터링
- 설명하지 못하는 구조 도입
- Issue 범위 밖 리팩토링
