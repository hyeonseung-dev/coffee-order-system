# Requirements

## 과제 목적

다중 서버 환경을 고려한 포인트 기반 커피 주문 시스템을 구현한다.

단순 CRUD 구현보다 데이터 정합성, 트랜잭션, 동시성 제어, 성능 개선, 테스트, 문서화를 설명 가능한 수준으로 정리하는 것이 목표다.

## 필수 기능

- 커피 메뉴 목록 조회
- 포인트 충전
- 커피 주문/결제
- 최근 7일 인기 메뉴 TOP 3 조회
- 주문 완료 후 후속 처리 흐름 분리

## 핵심 검증 항목

- 포인트 잔액이 음수가 되지 않아야 한다.
- 주문 실패 시 주문 데이터와 포인트 사용 이력이 저장되지 않아야 한다.
- 성공 주문만 인기 메뉴 집계 대상이 되어야 한다.
- 동일 사용자 동시 주문 정합성은 v1의 #10 문제 재현과 #11 DB 비관적 락 검증에서 확인한다.
- Redis는 원본 저장소가 아니라 캐시로만 사용해야 한다.

## 단계별 범위

### v0. 필수 API 완성

- 메뉴 조회
- 포인트 충전
- 주문/결제
- 최근 7일 인기 메뉴 조회
- 기본 예외 처리

### v1. 트랜잭션 / 동시성 검증

- 주문/결제 트랜잭션 경계 정리
- 포인트 차감 정합성 검증
- #10에서 동일 사용자 동시 주문 문제 재현
- #11에서 DB 비관적 락 적용과 정합성 검증

### v2. 성능 개선

- 인기 메뉴 조회 SQL 검증
- 인덱스 적용 전후 비교
- Redis 캐시 hit/miss 검증
- TTL 정책 확인

### v3. 선택 도전

- MySQL Replication
- Redis Master-Replica
- RabbitMQ 이벤트 분리

v3는 선택 도전이며 v0~v2 완료 이후 진행한다.
