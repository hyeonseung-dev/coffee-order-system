# Course Plan

## 현재 진도

- 챕터 1~3: 수강 완료
- 다음 수강 시작점: 챕터 4

## 강의 우선순위

1. 개인과제 v0~v2 제출 가능 상태 만들기
2. 과제와 직접 연결되는 챕터 4 이해
3. 필요한 부분만 실습
4. 챕터 5 부하 테스트를 프로젝트 성능 검증과 연결
5. 최종 프로젝트 대비 확장 학습

## 다음 수강 순서

### 챕터 4

1. 4-1 실시간 인기 상품 랭킹
2. 4-2 실시간 상품 결제 히스토리 기록
3. 4-3 결제 완료 기반 배송 흐름 구성하기
4. 4-4 에러 대응하기: 중복 방지, DLT

### 챕터 5

- 부하 테스트 개념
- K6 시작하기
- 이후 챕터는 개인과제 v0 진행 상황에 따라 수강

## 개인과제와 직접 연결되는 내용

- 인기 메뉴 집계와 랭킹
- 주문·결제 이력
- Commit 이후 후속 처리
- 중복 처리와 실패 대응
- 부하 테스트와 성능 측정
- K6

## 적용 원칙

강의 예제를 그대로 프로젝트에 붙이지 않는다.

- 현재 Issue가 해결하는 문제와 연결되는가?
- MySQL이 정확한 원본이라는 설계와 충돌하지 않는가?
- Redis·Kafka·DLT가 실제 요구사항에 필요한가?
- v0~v2 제출 일정에 부담을 주지 않는가?
- 구현보다 개념 이해가 우선인 내용인가?

현재 개인과제에서는 Kafka를 우선 구현하지 않는다. 챕터 4는 개념과 설계 비교 중심으로 듣고, 실제 구현은 현재 Issue 범위와 프로젝트 단계에 맞는 부분만 선택한다.
