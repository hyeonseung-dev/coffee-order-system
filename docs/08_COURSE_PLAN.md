# Course Plan

## 강의 우선순위

1. 개인과제 v0~v2 제출 가능 상태 만들기
2. 과제와 직접 연결되는 강의 이해
3. 필요한 부분만 실습
4. 최종 프로젝트 대비 확장 학습

## 과제와 직접 연결되는 강의

- 실시간 인기 상품 랭킹
- 결제 히스토리 기록
- 결제 완료 기반 후속 처리
- 에러 대응
- 부하 테스트
- K6

## 과제와 간접 연결되는 강의

- Kafka 기본 설정
- Kafka 커스텀 객체
- Offset과 Commit
- Kafka 에러 처리
- Kafka 운영 Best Practice

Kafka는 현재 개인과제의 우선 구현 대상이 아니다. v0~v2를 먼저 완료하고, Kafka는 최종 프로젝트 대비 학습 관점에서 정리한다.

## 실습 여부 판단 기준

- 현재 과제 단계와 직접 연결되는가?
- 오늘 작업 시간이 충분한가?
- 실습 결과를 README/docs/TIL에 남길 수 있는가?
- 제출 일정에 부담을 주지 않는가?
- 구현보다 개념 이해가 우선인 내용인가?

v0 진행 중에는 이해 위주로 보고, v2 이후 성능 측정이나 부하 테스트와 직접 연결될 때 실습을 검토한다.
