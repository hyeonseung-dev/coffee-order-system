## PR Issue #6 - 커피 메뉴 목록 조회 API

### 작업 내용

커피 메뉴 목록 조회 API를 구현하고, 구현 후 Human 검토와 ChatGPT 1차 리뷰를 진행했다.

초기 구현에서는 Controller가 `Map<String, List<MenuResponse>>` 형태로 응답을 구성하고 있었고, 테스트 코드 역시 준비·실행·검증 흐름이 명확하게 구분되지 않았다.

Human 검토를 통해 응답 타입 안정성과 테스트 가독성을 개선했으며, 이후 ChatGPT 리뷰에서 Issue 범위 관리, 환경변수 설정, 비밀번호 하드코딩 문제를 추가로 점검했다.

## PR Issue #7 - 포인트 충전 API

### 작업 내용

포인트 충전 API의 예외 처리 구조를 검토하고 리팩토링했다.

기존에는 `PointChargeController` 내부에 여러 `@ExceptionHandler`가 작성되어 있어 Controller가 정상 요청 처리와 예외 응답 처리를 함께 담당하고 있었다.

1차 리뷰에서는 Controller의 예외 처리 로직을 `GlobalExceptionHandler`로 이동하는 수준으로 범위를 제한했다.

이후 개별 비즈니스 예외 클래스가 기능 추가에 따라 계속 증가할 수 있다는 점을 재검토했고, 비즈니스 예외는 `BusinessException` 하나로 통합하고 에러 정보는 `ErrorCode enum`에서 관리하는 구조를 적용하기로 결정했다.

단, 스프링 프레임워크가 직접 발생시키는 검증 및 JSON 파싱 예외는 `BusinessException`으로 변환하지 않고 `GlobalExceptionHandler`에서 별도로 처리한다.
